# GitignoreManager

Automatic .gitignore management for Rulesets with flexible override mechanisms.

## Overview

GitignoreManager automatically manages `.gitignore` entries for files generated by Rulesets compilation. It provides a robust system with override mechanisms to give you complete control over what gets ignored or kept in version control.

## Features

- **Automatic .gitignore Management**: Automatically adds generated file paths to `.gitignore`
- **Managed Comment Blocks**: Uses `# START Rulesets Generated Files` and `# END Rulesets Generated Files` comments to manage entries
- **Override Files Support**: `.rulesetkeep` and `.rulesetignore` files for fine-grained control
- **Configuration Support**: Programmatic configuration via config objects
- **Path Normalization**: POSIX-style relative paths for cross-platform compatibility
- **Error Handling**: Graceful error handling without breaking compilation
- **Non-Intrusive**: Preserves existing .gitignore content outside managed blocks

## Quick Start

### Basic Usage

GitignoreManager is automatically integrated into the Rulesets compilation process:

```typescript
import { runRulesetsV0 } from '@rulesets/core';

// GitignoreManager runs automatically during compilation
await runRulesetsV0('./my-rules.ruleset.md');
// Generated files are automatically added to .gitignore
```

### Manual Usage

```typescript
import { createGitignoreManager } from '@rulesets/core';

const gitignoreManager = createGitignoreManager();
const result = await gitignoreManager.updateGitignore([
  '.cursor/rules/my-rules.mdc',
  '.windsurf/rules/my-rules.md',
]);

console.log(`Added ${result.added.length} files to .gitignore`);
```

## Override Files

### .rulesetkeep - Keep Files in Git

Create a `.rulesetkeep` file to specify patterns that should NOT be ignored:

```
# Keep important cursor rules in git
.cursor/rules/important.mdc
.cursor/rules/shared-*

# Keep windsurf documentation rules
.windsurf/rules/docs.md
```

### .rulesetignore - Additional Ignore Patterns

Create a `.rulesetignore` file to specify additional patterns to ignore:

```
# Ignore temporary files
*.tmp
*.temp

# Ignore debug outputs
debug/
logs/
```

## Configuration

### Programmatic Configuration

```typescript
import { GitignoreManager } from '@rulesets/core';

const manager = new GitignoreManager({
  enabled: true,
  commentPrefix: 'MyTool',
  alwaysIgnore: ['*.log', 'temp/'],
  alwaysKeep: ['important.txt'],
});
```

### Configuration Options

| Option          | Type       | Default      | Description                              |
| --------------- | ---------- | ------------ | ---------------------------------------- |
| `enabled`       | `boolean`  | `true`       | Enable/disable gitignore management      |
| `commentPrefix` | `string`   | `'Rulesets'` | Custom prefix for managed block comments |
| `alwaysIgnore`  | `string[]` | `[]`         | Patterns to always ignore                |
| `alwaysKeep`    | `string[]` | `[]`         | Patterns to always keep                  |

## Generated .gitignore Format

GitignoreManager creates managed blocks in your `.gitignore`:

```gitignore
# Existing content is preserved
node_modules/
dist/

# START Rulesets Generated Files
# Generated by Rulesets - do not edit manually
.cursor/rules/my-rules.mdc
.windsurf/rules/my-rules.md
# END Rulesets Generated Files

# More existing content
*.log
```

## Override Priority

Override rules are applied in the following priority order (highest to lowest):

1. **Keep patterns** (`.rulesetkeep` + `alwaysKeep` config) - Never ignored
2. **Ignore patterns** (`.rulesetignore` + `alwaysIgnore` config) - Always ignored
3. **Default behavior** - Generated files are ignored

## Path Handling

GitignoreManager normalizes all paths to:

- **Relative paths** from project root
- **POSIX-style separators** (`/` instead of `\`)
- **No leading `./`** (e.g., `src/file.ts` not `./src/file.ts`)

## Error Handling

GitignoreManager is designed to fail gracefully:

- **Non-critical errors** are logged as warnings
- **Compilation continues** even if gitignore updates fail
- **Invalid patterns** are skipped with warnings
- **Permission errors** don't break the build process

## API Reference

### GitignoreManager

```typescript
class GitignoreManager {
  constructor(config?: Partial<GitignoreConfig>, basePath?: string);

  updateGitignore(generatedPaths: readonly string[]): Promise<GitignoreResult>;
  readOverrides(): Promise<GitignoreOverrides>;
  shouldIgnore(path: string): boolean;
  isEnabled(): boolean;
  clearCache(): void;
}
```

### Factory Function

```typescript
function createGitignoreManager(
  config?: Partial<GitignoreConfig>,
  basePath?: string
): GitignoreManager;
```

### Types

```typescript
interface GitignoreConfig {
  readonly enabled: boolean;
  readonly commentPrefix?: string;
  readonly alwaysIgnore?: readonly string[];
  readonly alwaysKeep?: readonly string[];
}

interface GitignoreResult {
  readonly success: boolean;
  readonly added: readonly string[];
  readonly kept: readonly string[];
  readonly messages: readonly string[];
}

interface GitignoreOverrides {
  readonly keep: readonly string[];
  readonly ignore: readonly string[];
  readonly config: readonly string[];
}
```

## Advanced Usage

### Custom Comment Prefix

```typescript
const manager = new GitignoreManager({
  commentPrefix: 'MyCompiler',
});
// Generates: # START MyCompiler Generated Files
```

### Disable for Specific Projects

```typescript
const manager = new GitignoreManager({
  enabled: false,
});
// No gitignore operations will be performed
```

### Complex Override Patterns

**.rulesetkeep**:

```
# Keep all cursor rules
.cursor/rules/*

# Keep specific windsurf files
.windsurf/rules/important.md
.windsurf/rules/shared-*.md
```

**.rulesetignore**:

```
# Ignore all temporary files
*.tmp

# Ignore debug directories
debug/
logs/

# Ignore large generated files
*.bundle.*
```

## Troubleshooting

### Common Issues

1. **Files not being ignored**: Check override files (`.rulesetkeep`) that might be keeping them
2. **Permission errors**: Ensure write access to project root for `.gitignore`
3. **Paths not normalized**: GitignoreManager automatically handles path normalization
4. **Managed block corrupted**: Delete the managed block and re-run compilation

### Debug Logging

Enable debug logging to see what GitignoreManager is doing:

```typescript
import { ConsoleLogger } from '@rulesets/core';

const logger = new ConsoleLogger();
// Set logger level to debug in your logging configuration
await runRulesetsV0('./my-rules.ruleset.md', logger);
```
