# Amp Provider - Rules Usage Guide

The Amp provider compiles your source rules into `AGENT.md` format for use with Amp AI assistant.

## Overview

**Provider ID**: `amp`  
**Output Format**: Markdown  
**Output File**: `AGENT.md` (singular, as per Amp's specification)  
**Provider Type**: Agent  

## Features

- ✅ Simple Markdown output
- ✅ Automatic formatting optimization
- ✅ Syntax highlighting support
- ✅ Block-based organization
- ✅ Import system support
- ✅ Variable substitution
- ❌ XML output (not supported by Amp)
- ❌ Complex MCP configuration (not needed)

## Configuration

### Basic Configuration

Add Amp to your `ruleset.config.json`:

```json
{
  "providers": {
    "amp": {
      "enabled": true
    }
  }
}
```

### Advanced Configuration

```json
{
  "providers": {
    "amp": {
      "enabled": true,
      "outputPath": "AGENT.md",
      "priority": "high",
      "includeProjectContext": true
    }
  }
}
```

### Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | `false` | Enable/disable Amp provider |
| `outputPath` | string | `"AGENT.md"` | Output file path (should be AGENT.md) |
| `priority` | string | `"medium"` | Priority level: `low`, `medium`, `high` |
| `includeProjectContext` | boolean | `true` | Include project-specific context |

## Output Format

The Amp provider generates clean Markdown optimized for AI assistant consumption:

### Input (source rules)
```markdown
{{instructions}}
Follow these coding standards:
- Use TypeScript for all new code
- Write comprehensive tests
- Document all public APIs
{{/instructions}}

{{examples}}
```typescript
// Good: Clear function name and types
function calculateTotal(items: CartItem[]): number {
  return items.reduce((sum, item) => sum + item.price, 0);
}
```
{{/examples}}
```

### Output (AGENT.md)
```markdown
# Instructions

Follow these coding standards:
- Use TypeScript for all new code
- Write comprehensive tests
- Document all public APIs

# Examples

```typescript
// Good: Clear function name and types
function calculateTotal(items: CartItem[]): number {
  return items.reduce((sum, item) => sum + item.price, 0);
}
```
```

## Block Support

The Amp provider supports all standard Rulesets blocks:

- `{{instructions}}` - Main guidelines and rules
- `{{examples}}` - Code examples and patterns
- `{{requirements}}` - Project requirements
- `{{guidelines}}` - Best practices
- `{{context}}` - Background information

## Import System

Use imports to share common rules:

```markdown
{{> @common/typescript-standards}}
{{> project-specific#coding-style}}
```

## Variable Substitution

Variables are automatically resolved:

```markdown
Project: {{$project.name}}
Version: {{$project.version}}
AI Assistant: {{$destination.name}}
```

## Best Practices

### 1. Keep It Simple
Amp prefers clean, straightforward Markdown without complex formatting:

```markdown
<!-- ✅ Good: Simple and clear -->
{{instructions}}
Use consistent naming conventions throughout the codebase.
{{/instructions}}

<!-- ❌ Avoid: Complex formatting -->
{{instructions}}
<div class="important-notice">
  <strong>CRITICAL:</strong> Use consistent naming conventions!
</div>
{{/instructions}}
```

### 2. Use Descriptive Block Names
```markdown
<!-- ✅ Good: Descriptive names -->
{{typescript-guidelines}}
{{error-handling-patterns}}
{{testing-requirements}}

<!-- ❌ Avoid: Generic names -->
{{block1}}
{{stuff}}
{{misc}}
```

### 3. Organize Content Logically
Structure your rules from general to specific:

```markdown
{{project-overview}}
{{coding-standards}}
{{typescript-specific}}
{{testing-guidelines}}
{{deployment-notes}}
```

## Example Rule File

Create `.ruleset/src/amp-rules.rule.md`:

```markdown
---
description: "AI coding assistant rules for Amp"
destination:
  include: ["amp"]
---

# AI Assistant Rules

{{instructions}}
You are a helpful coding assistant for this TypeScript project.

## Core Principles
- Write clean, maintainable code
- Follow TypeScript best practices
- Prioritize code safety and readability
- Use meaningful variable and function names

## Code Style
- Use 2-space indentation
- Prefer `const` over `let` when possible
- Use explicit return types for functions
- Add JSDoc comments for public APIs
{{/instructions}}

{{examples}}
```typescript
// ✅ Good: Explicit types and clear naming
interface User {
  id: string;
  email: string;
  createdAt: Date;
}

async function getUserById(id: string): Promise<User | null> {
  return await database.user.findUnique({ where: { id } });
}
```

```typescript
// ❌ Avoid: Implicit types and unclear naming
async function get(id) {
  return await db.user.findUnique({ where: { id } });
}
```
{{/examples}}

{{context}}
This is a modern TypeScript web application using:
- Node.js runtime
- TypeScript for type safety
- Express.js for API routes
- Prisma for database access
- Jest for testing
{{/context}}
```

## Compilation Command

Compile your rules for Amp:

```bash
# Compile all providers
rulesets compile

# Compile only Amp
rulesets compile --provider amp

# Watch mode for development
rulesets compile --watch --provider amp
```

## Integration with Amp

After compilation, your `AGENT.md` file will be automatically recognized by Amp:

1. Amp reads `AGENT.md` from your project root
2. Rules are applied to all AI interactions within the project
3. Context is automatically provided to improve assistance quality

## Troubleshooting

### Common Issues

**Issue**: Empty `AGENT.md` file
**Solution**: Check that your source rules have content and blocks are properly formatted

**Issue**: Rules not being applied in Amp
**Solution**: Ensure `AGENT.md` is in your project root and Amp has access to read it

**Issue**: Compilation errors
**Solution**: Validate your rule syntax and check for proper block closing tags

### Debug Mode

Enable debug mode for detailed compilation information:

```json
{
  "debug": true,
  "providers": {
    "amp": {
      "enabled": true
    }
  }
}
```

## Migration from Other Providers

### From Claude Code

Claude Code uses `CLAUDE.md` while Amp uses `AGENT.md`:

```json
{
  "providers": {
    "claude-code": {
      "enabled": false
    },
    "amp": {
      "enabled": true
    }
  }
}
```

### Content Differences

| Feature | Claude Code | Amp |
|---------|-------------|-----|
| Output file | `CLAUDE.md` | `AGENT.md` |
| XML support | ✅ | ❌ |
| MCP config | ✅ | ❌ |
| Markdown | ✅ | ✅ |
| Block system | ✅ | ✅ |
